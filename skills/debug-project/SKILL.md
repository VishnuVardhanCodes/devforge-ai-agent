# Debug Project Skill

## Purpose

This skill analyzes existing codebases and fixes errors.

It detects runtime errors, logical bugs, and configuration issues.

It improves project reliability and usability.

---

## Supported Debug Types

DevForge must support fixing:

* Syntax errors
* Runtime errors
* Import/module errors
* Dependency issues
* File path errors
* Configuration mistakes
* Broken APIs
* Missing files
* Logic bugs

---

## Execution Workflow

When triggered:

### Step 1 — Inspect Project

Read all relevant project files.

Focus on:

* Source files
* Config files
* Package files
* Error logs

---

### Step 2 — Identify Issues

Detect:

* Syntax problems
* Missing dependencies
* Incorrect imports
* Misconfigured settings
* Runtime exceptions

---

### Step 3 — Determine Fix Strategy

Choose:

* Code correction
* Dependency update
* File restructuring
* Configuration fixes

---

### Step 4 — Apply Fixes

Modify:

* Source code
* Config files
* Dependencies
* Import statements

Ensure correctness.

---

### Step 5 — Validate Fix

Confirm:

* Errors resolved
* Code runs logically
* Structure remains valid

---

### Step 6 — Document Fix

Create update notes:

Include:

* What was broken
* What was fixed
* Why it works now

---

## Output Requirements

DevForge must:

* Fix errors logically
* Maintain code readability
* Avoid introducing new bugs
* Preserve project structure
* Explain fixes clearly

---

## Quality Standards

DevForge must always:

* Diagnose before fixing
* Apply minimal changes
* Maintain clean code
* Document debugging steps
