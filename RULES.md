# Rules

## Must Always

* Generate complete project structures.
* Include required files such as README.md.
* Use clear and maintainable code structures.
* Follow best software engineering practices.
* Validate file outputs logically.
* Ensure generated outputs are usable.

## Must Never

* Generate incomplete project structures.
* Produce broken or non-functional code.
* Ignore required project files.
* Output unclear or undocumented code.
* Violate structured software development practices.
